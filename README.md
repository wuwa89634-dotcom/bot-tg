# MangaBuff Club Telegram Bot

Бот для клуба MangaBuff: регистрация участников по ссылке на профиль, проверка членства в клубе, меню, запись на вклады на следующие 48 часов, розыгрыши и список зарегистрированных участников.

## Быстрый запуск

1. Создайте бота у `@BotFather` и получите токен.
2. Установите Python 3.11+.
3. Установите зависимости:

```bash
pip install -r requirements.txt
```

4. Скопируйте `.env.example` в `.env` и заполните `BOT_TOKEN`.
   Для Railway укажите `CLUB_URL` ссылкой на нужный клуб, например `https://mangabuff.ru/clubs/fu-razvrat`.
   `CHAT_URL` можно заполнить ссылкой на чат или тему, где будет работать бот.
   `ADMIN_IDS` заполняется Telegram ID админов через запятую; админ-функции будут добавлены позже.
5. Запустите:

```bash
python bot.py
```

## Railway и PostgreSQL

На Railway подключите PostgreSQL к сервису с ботом. Railway автоматически добавит переменную `DATABASE_URL`; бот увидит ее и будет хранить пользователей, брони, настройки табло и базу участников чата в PostgreSQL.

Для локального запуска без `DATABASE_URL` бот использует SQLite-файл из `DB_PATH`.

## Как работает регистрация

При первом `/start` бот просит ссылку вида:

```text
https://mangabuff.ru/users/854887
```

Затем бот:

- достает ID профиля;
- проверяет, не зарегистрирован ли уже этот Telegram-пользователь;
- проверяет, не привязан ли уже этот MangaBuff-профиль к другому Telegram-пользователю;
- открывает страницу профиля, чтобы взять имя;
- открывает страницу клуба `CLUB_URL`;
- ищет ID профиля в блоке участников `.club__members`.

Проверка клуба находится в `mangabuff.py`. По присланному HTML участники читаются из ссылок вида `/users/854887` внутри `.club__member-name`.

Если MangaBuff не показывает список участников без входа в аккаунт, есть два варианта:

- добавить `MANGABUFF_COOKIE` со значением cookie авторизованной сессии MangaBuff;
- или добавить `MANGABUFF_EMAIL` и `MANGABUFF_PASSWORD`, чтобы бот сам входил в аккаунт MangaBuff.

Для автоматического входа по умолчанию используется `MANGABUFF_LOGIN_URL=https://mangabuff.ru/login`, поле логина `email`, поле пароля `password`. Если форма MangaBuff использует другие имена полей, их можно переопределить через `MANGABUFF_LOGIN_FIELD` и `MANGABUFF_PASSWORD_FIELD`.

Если в Railway logs появилась причина `login_failed`, попробуйте заменить `MANGABUFF_LOGIN_FIELD` на `login` или `username`.

Если MangaBuff возвращает `403` уже на `https://mangabuff.ru/login`, значит сайт блокирует запросы с Railway до авторизации. В этом случае можно задать `MANGABUFF_PROXY_URL`, чтобы запросы к MangaBuff шли через другой адрес, или перенести бота на сервер, который MangaBuff не блокирует.

Если регистрация не проходит из-за доступа к MangaBuff, откройте Railway logs и найдите строку `MangaBuff profile check failed`: там будет причина (`auth_required`, `club_not_found`, `members_unavailable`, `network`) и технические детали.

## Расписание в чате

Если бот добавлен в чат клуба:

1. Добавьте бота в чат.
2. Если чат разделен на темы, откройте нужную тему для записей.
3. В этой теме отправьте `/post_schedule`.
4. Бот создаст одно сообщение с расписанием и сохранит ID чата, темы и сообщения.
5. После новых броней бот будет редактировать это же сообщение, а не отправлять новые.

Повторный `/post_schedule` не создает дубль: бот обновит уже сохраненное сообщение. Если команду отправить в другой теме, бот попробует удалить старое табло и перенести расписание в текущую тему.

Если нужно указать сообщение вручную, можно заполнить `GROUP_CHAT_ID`, `SCHEDULE_MESSAGE_ID` и `SCHEDULE_THREAD_ID` в `.env`.

## Команды в чате

- `.ник` - ответьте этой командой на сообщение участника, и бот пришлет ссылку на его профиль MangaBuff.
- `.распес` - бот пришлет актуальное расписание вкладов.
- `.всем текст` - бот повторит текст и отметит всех известных участников чата.
- `/set_giveaway_topic` - назначить текущую тему для публикации розыгрышей. Команду может использовать администратор чата или ID из `ADMIN_IDS`.

Для `.всем` бот ведет базу участников каждого чата: запоминает людей по сообщениям, входам в чат и ответам на `.ник`, а при событии выхода удаляет человека из базы. Это тот же принцип, на котором обычно работают боты с массовыми отметками.

## Розыгрыши

1. Администратор отправляет `/set_giveaway_topic` в теме, где должны публиковаться розыгрыши.
2. Зарегистрированный участник открывает `Розыгрыши` в главном меню и нажимает `Создать розыгрыш`.
3. Бот по очереди запрашивает название, фото/GIF, описание, количество победителей и дату окончания в формате `10.10.26 10:10`.
4. После предпросмотра автор публикует розыгрыш в назначенную тему.
5. Кнопка `Участвовать` принимает только зарегистрированных пользователей и защищена от повторного участия.
6. В назначенное время бот случайно выбирает победителей, удаляет кнопку и ответом на публикацию отмечает победителей.

Автор и администраторы из `ADMIN_IDS` могут отменить активный розыгрыш. Активные и завершенные розыгрыши сохраняются в PostgreSQL, включая участников и победителей.

## Что уже есть

- регистрация по профилю MangaBuff;
- защита от повторной регистрации одного профиля;
- главное меню с фото/текстом из `.env`;
- запись на следующие 48 часов по московскому времени;
- прошедшие слоты скрываются;
- занятые слоты показываются с крестиком;
- повторное нажатие на свою бронь отменяет ее;
- создание и автоматическое завершение розыгрышей;
- список зарегистрированных участников.
